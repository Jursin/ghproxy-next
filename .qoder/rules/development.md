## 代码实现指南

### 代码风格与结构
- 编写简洁、技术性的 TypeScript 代码，并提供准确的示例。
- 使用函数式和声明式编程模式；避免使用类。
- 优先选择迭代和模块化，遵循DRY原则，避免代码重复。
- 优先考虑性能优化和最小化JavaScript以获得最佳用户体验。
- 编写语义化的HTML以提高可访问性和SEO。
- 必须添加函数级的中文注释。
- 相同区域或模块的代码尽量在文件的同一位置，以功能进行代码的分块，方便后期审计和优化。

### 命名约定
- 目录使用小写字母和短横线（例如，`components/auth-wizard`）。
- 使用反映业务逻辑和功能角色的描述性变量和函数/常量名称。事件函数应以 `handle` 为前缀命名，例如 `onClick` 的 `handleClick` 和 `onKeyDown` 的 `handleKeyDown`。
- 优先使用命名导出组件。

### 语法与格式化
- 对于纯函数，使用`function`关键字以利用提升和清晰性。
- 条件语句中避免不必要的大括号；简单语句使用简洁语法。
- 使用Prettier保持代码格式一致。

### UI与样式
- 使用Shadcn Vue、Radix Vue和Tailwind进行组件和样式设计。
- 使用Tailwind CSS实现响应式设计；采用移动优先的方法。
- 始终使用 Tailwind 类来为 HTML 元素添加样式；避免使用 CSS 或标签。
- 确保响应式设计，考虑不同的屏幕尺寸和方向。
- 在类标签中尽可能使用 `class:` 而不是三元运算符。

### 性能优化
- 对非关键组件使用动态加载。
- 优化图片：使用 WebP 格式，包含尺寸信息，实现懒加载。

### 安全性
- 对用户输入进行清理以防止XSS攻击。
